# Instructor Guide

This course is designed to run inside a community, not as a one-off class.

## Delivery Principles

- Make it visual first. Then make it practical. Then make it explainable.
- Every session ends with a “proof artifact” learners can share.
- Treat errors as normal: debugging is a core skill, not a failure.
- Keep the first 3 weeks gentle. People quit early if they feel stupid.

## Session Structure (90 minutes)

1. Warmup (5 min): recap + one small question
2. Concept (15 min): one idea, one diagram, one analogy
3. Demo (15 min): show the idea in reality
4. Lab (40 min): learners do the thing, instructor circulates
5. Share (10 min): 2–3 learners show output
6. Next step (5 min): what to do before next session

## Weekly Community Loop

- Week starts: “Focus of the week” post + 7-day sprint checklist
- Mid-week: “Blockers thread” post (ask one focused question)
- End-week: “Proof day” showcase (screenshots, repo link, short demo)

Suggested templates:

- `templates/DISCORD_WEEKLY_POST.md`
- `templates/PROOF_OF_WORK_TEMPLATE.md`

## Assessment Philosophy

We grade outcomes, not vibes:

- Can you explain it in plain language?
- Can you demonstrate it working?
- Can you document it so someone else can run it?

See `ASSESSMENT_RUBRICS.md`.

## Hardware/Software Requirements

Minimum:

- Any Windows/macOS/Linux machine
- A browser
- A code editor (VS Code recommended)
- Python installed (for the programming module)

Optional (for deeper learning):

- A Raspberry Pi or microcontroller kit
- A spare laptop/desktop to open and inspect components

## Risk Management (Dropout Prevention)

Common failure points:

- Week 1–2: “I’m not smart enough” anxiety
- Week 8–10: environment setup issues
- Week 14–16: logic and loops feel abstract

Countermeasures:

- Pair learners (buddy system)
- Provide “known-good setup” checklist
- Keep lab exercises tiny and repeatable

## Instructor Checklist (Before Each Week)

- Decide the single “core concept” for the week
- Prepare the diagram and demo
- Prepare the lab worksheet
- Define the deliverable format (file names, repo structure, screenshots)
