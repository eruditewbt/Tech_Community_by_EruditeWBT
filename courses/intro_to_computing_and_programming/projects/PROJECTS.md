# Projects (Outcome-Based)

These are designed to produce “proof of work” quickly.

## Mini-Project 1 (Week 18)

Pick one:

- File organizer: sort downloads into folders by type; write a log file.
- Simple dataset report: load CSV/JSON, compute 3 metrics, output a report.

Proof requirements:

- README with run steps
- screenshot of output
- limitations + next step

## Mini-Project 2 (Week 22–24)

Pick one:

- Personal developer dashboard (local): show your tools, notes, and links; generate a report.
- Quiz + scoring system: categories, randomized questions, saved results.

Proof requirements:

- diagram of the system (inputs → steps → outputs)
- demo recording (30–60 seconds)

## Capstone (Week 22–24, v1)

Choose a real-life problem and build a small system.

Examples:

- Study tracker (inputs: study sessions; outputs: weekly report)
- Budget tracker (inputs: expenses; outputs: summary + charts)
- Community directory (inputs: profiles; outputs: search + export)

Capstone proof requirements:

- problem statement (one page)
- working MVP
- README + screenshots + limitations
