# Weekly System
*Powered by EruditeWBT*

A community becomes “real” when it has rhythm.

## Weekly Sprint (Simple)
1. Pick 1 project task for the week.
2. Build for 2–6 hours total.
3. Ship one update:
   - screenshot or short demo
   - short write-up
   - diagram (even rough)
4. Ask one question. Help one person.

## Proof Rule
Every week should create at least one of:
- a demo
- a diagram
- a tutorial note
- a reusable component

Use the repo templates:
- `templates/PROJECT_PITCH_TEMPLATE.md`
- `templates/PROJECT_REVIEW_TEMPLATE.md`
